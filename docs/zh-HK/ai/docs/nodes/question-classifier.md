# 问题分类器（Question Classifier）

## 是什麼

问题分类器用于**判斷用户输入属于哪一类问题**，并根据分类结果将流程导向不同分支。

它的目标不是生成内容，而是**做出明確判斷**。

## 您可以用它做什麼

#### 1. 识别用户问题类型

问题分类器可以帮助你判斷：

* 用户在问什么类型的问题
* 当前输入适合走哪条处理路径

例如：

* 咨询类 / 操作类 / 闲聊类
* 技术问题 / 业务问题 / 无效输入

---

#### 2. 驱动流程分流

分类结果通常会：

* 直接连接到不同下游節點
* 决定调用不同模型、Agent 或工具

它是構建**多路流程**的核心節點之一。

## 輸出结果

* 輸出为**明確的分类标签**
* 每个分类结果对应一条分支路径
* 分类结果可被条件節點或直接用于分支连接

## 使用規則

* 分类类别需要提前定义
* 每次输入只能命中一个分类结果
* 不用于生成最终回答

## 使用建議

* 分类数量不宜过多（建议 3～7 类）
* 每一类边界要清晰、互不重叠
* 兜底分类非常重要（如“其他”）

## 典型場景

* 智能客服分流
* 對話入口意图判斷
* 不同业务流程的路由

## 什么时候不该用？

* 不需要分流流程
* 分类标准不明確
* 只是想生成回覆内容

这类场景更适合直接使用 LLM 節點。
