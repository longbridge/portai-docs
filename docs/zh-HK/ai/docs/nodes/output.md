# 结束節點（End）

## 是什麼

结束節點用于 Workflow 的 **标记流程的完成位置**。
当流程執行到结束節點时，整个流程会停止运行，并将最终结果返回给调用方或用户。

这是流程中的**终点節點**。

## 您可以用它做什麼

#### 1. 结束流程執行

结束節點用于明確告诉系统：

* 当前流程已经完成
* 不再继续執行后续節點

无论流程多複雜，最终都应以结束節點作為收尾。

---

#### 2. 返回最终輸出结果

你可以在结束節點中：

* 选择需要返回的变量或内容
* 将上游節點（如大模型節點）的輸出作為最终结果返回

例如：

* 返回模型生成的文本
* 返回结构化数据（如 JSON）
* 返回经过多步处理后的结果

---

#### 3. 统一流程出口

在包含条件分支的流程中：

* 不同分支可以连接到同一个结束節點
* 用于统一流程的最终輸出和结束行为

## 輸出说明

* 结束節點的輸出即为**流程的最终輸出**
* 返回内容会：

  * 展示给用户
  * 或作為 API 调用的响应结果
* 如果未配置返回内容，流程将只執行但不返回数据

## 使用規則

* 一个流程中**至少需要一个结束節點**
* 结束節點：

  * 不能连接下游節點
  * 只能作為流程的终点
* 一个流程中可以存在**多个结束節點**（用于不同分支返回不同结果）

## 使用建議

* 明確指定你希望返回的内容，避免返回无关变量
* 对外提供 API 的流程，建议返回结构清晰、稳定的数据格式
* 多分支流程中，确保每个分支最终都能走到某个结束節點

## 典型場景

* 返回對話回覆给用户
* 返回 Agent 執行结果
* 返回工具或模型处理后的最终数据
* 作為子流程的结束出口
