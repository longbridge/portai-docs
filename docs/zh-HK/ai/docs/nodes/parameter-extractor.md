# 参数提取器（Parameter Extractor）

## 是什麼

参数提取器用于**从自然语言输入中提取结构化参数**，并輸出为可直接使用的变量。

它的目标是：

> **把“人说的话”，变成“流程能用的数据”。**

## 您可以用它做什麼

#### 1. 从文本中提取关键信息

参数提取器可以帮助你提取：

* 时间、地点、数量
* 用户指定的条件或对象
* 執行任務所需的参数

例如：

* “帮我查一下明天北京的天气”

  * 时间：明天
  * 地点：北京

---

#### 2. 为后续節點准备结构化输入

提取出的参数可以直接用于：

* HTTP 请求節點
* 工具節點
* Agent 節點
* 条件判斷節點

## 輸出结果

* 輸出为结构化字段（参数名 + 参数值）
* 未识别的参数可为空或使用默认值
* 可直接在流程中引用

## 使用規則

* 参数名称需要提前定义
* 每个参数应有清晰语义
* 提取器只做提取，不做业务判斷

## 使用建議

* 参数数量不要过多
* 参数定义尽量具体，避免歧义
* 对关键参数做好校验或兜底处理

## 典型場景

* 自然语言转 API 调用
* 對話式任務執行
* 结构化请求生成

## 什么时候不该用？

* 输入本身已经是结构化数据
* 不需要提取具体参数
* 任務更偏向开放式生成


## 问题分类器 vs 参数提取器（快速理解）

| 对比项      | 问题分类器     | 参数提取器     |
| -------- | --------- | --------- |
| 主要目的     | 判斷“属于哪一类” | 提取“有哪些参数” |
| 輸出类型     | 分类标签      | 结构化字段     |
| 是否用于分流   | 是       | 否       |
| 是否用于執行准备 | 间接     | 直接      |
| 常见下游     | 分支流程      | 工具 / API  |

## 組合使用示例

常见流程模式是：

1. **问题分类器** → 判斷问题类型
2. **参数提取器** → 提取该类型所需参数
3. **工具 / Agent** → 執行具体任務

## 一句话记住

* **不知道“走哪条路”** → 用 **问题分类器**
* **知道要做什么，但缺参数** → 用 **参数提取器**
