# 变量赋值器（Variable Assigner）

## 是什麼

变量赋值器用于**創建、修改或覆盖流程中的变量值**。
它让你可以在流程執行过程中，**显式地控制变量内容**，而不是依赖模型或工具的隐式輸出。

你可以把它理解为：

> **“在流程中手动设置或更新变量的地方。”**

## 您可以用它做什麼

#### 1. 創建新变量

你可以使用变量赋值器：

* 定义新的流程变量
* 为变量设置初始值或默认值
* 将计算结果保存为可复用的数据

例如：

* 设置状态标识
* 初始化计数器
* 生成中间结果变量

---

#### 2. 修改已有变量

变量赋值器可以：

* 覆盖上游節點产生的变量
* 基于已有变量生成新值
* 在不同分支中设置不同变量状态

这在条件分支或循环中非常常见。

---

#### 3. 控制流程状态

透過变量赋值器，你可以：

* 标记流程当前状态
* 控制循环或条件判斷的依据
* 在流程中显式表达“进度”或“结果状态”

## 输入与輸出

* **输入**：

  * 可引用任意已有流程变量
* **輸出**：

  * 更新后的变量集合
  * 变量可直接被后续節點使用

## 使用規則

* 变量名必须唯一（同一作用域内）
* 新赋值会覆盖同名变量的旧值
* 赋值器本身不进行複雜邏輯推理

## 使用建議

* 用变量赋值器做**明確、可控的状态修改**
* 避免在模型輸出中隐式改变关键变量
* 关键流程变量建议集中在赋值器中修改，方便排查问题
* 变量命名尽量语义清晰，避免歧义

## 典型場景

* 设置流程執行状态（如 success / failed）
* 在条件分支中标记不同结果
* 循环中更新计数或累计结果
* 将複雜節點輸出整理为统一变量

## 什么时候不该用？

* 只需要临时使用某个節點輸出
* 不关心变量的长期复用
* 需要複雜计算或推理（应使用 Code 或 LLM）

## 与其他節點的区别（快速理解）

| 節點      | 主要作用        |
| ------- | ----------- |
| 变量赋值器   | 显式设置 / 修改变量 |
| Code 節點 | 執行确定性计算     |
| LLM 節點  | 推理与生成       |
| 参数提取器   | 从文本中提取变量    |

## 一句话记住

> **变量赋值器不是“计算”，而是“控制”。**
> 用它来让流程的状态和数据变化**看得见、管得住**。
