# 開始

## 是什麼

開始節點是**流程的起点**。
当流程被触发时，系统会从这里開始執行，并把你提供的输入内容传递给后续節點。

每个流程**必须**包含一个開始節點。

## 您可以用它做什麼

#### 1. 接收流程输入

開始節點用于接收流程运行时的输入内容，例如：

* 用户输入的文本
* 介面调用时传入的参数
* 上层系统传递的数据

这些输入会作為流程的初始数据，供后续所有節點使用。

---

#### 2. 定义流程可用的变量

你可以在開始節點中：

* 定义流程的输入字段
* 设置字段名称和类型
* 作為后续節點引用的数据来源

例如：

* `query`：用户的问题
* `user_id`：用户标识
* `context`：上下文信息

---

#### 3. 触发整个流程运行

当流程被调用或執行时：

* 系统会从開始節點启动
* 按照畫布上的连接顺序，依次執行后续節點
* 開始節點本身不进行处理，只负责“開始”和“传递”

## 使用規則

* 一个流程中**只能有一个開始節點**
* 開始節點：

  * 不能被其他節點连接
  * 必须连接至少一个下游節點
* 如果没有開始節點，流程无法运行

## 使用建議

* 将**所有流程必需的输入**都放在開始節點中定义
* 字段命名尽量清晰，方便后续節點理解和引用
* 不要在開始節點中承担邏輯处理，它只负责输入和启动

## 典型場景

* 對話流程的用户输入入口
* Agent 執行前的参数配置
* API 流程的请求参数起点
* 子流程的统一入口
