# 大模型節點（LLM）

## 是什麼

大模型節點用于**调用 AI 大模型完成推理、生成或分析任務**。
你可以在这里編寫提示词（Prompt），并使用流程中的变量，让模型生成你需要的结果。

这是流程中**真正“思考”和“产出内容”**的節點。

## 您可以用它做什麼

#### 1. 生成文本内容

使用大模型節點，你可以让模型：

* 回答问题
* 撰写文案、总结内容
* 生成结构化文本
* 进行角色扮演式回覆

例如：

* 问答助手
* 内容生成器
* 智能客服回覆

---

#### 2. 理解和分析输入内容

你可以让模型对输入数据进行分析，例如：

* 理解用户意图
* 提取关键信息
* 分类、判斷或评分
* 转换输入格式

---

#### 3. 使用流程变量構建动态 Prompt

在大模型節點中，你可以：

* 引用開始節點或上游節點产生的变量
* 将变量插入到 Prompt 中
* 構建动态、可复用的提示词

例如：

```text
用户问题：{{query}}
请根据以上内容给出清晰、简洁的回答。
```

## 核心配置說明

#### Prompt（提示词）

* 用自然语言描述你希望模型完成的任務
* 支援引用流程变量
* 建议清晰描述角色、目标和輸出要求

---

#### 模型参数（如支援）

根据平台配置，你可能可以调整：

* 使用的模型类型
* 輸出长度
* 回覆风格或稳定性相关参数

这些参数会直接影响生成结果的表现。

## 輸出结果

* 大模型生成的内容会作為**節點輸出**
* 輸出结果可以：

  * 被后续節點继续处理
  * 作為最终结果返回给用户
  * 用于条件判斷或工具调用

## 使用規則

* 大模型節點必须连接在開始節點或其他節點之后
* 每次執行都会基于当前流程上下文重新生成结果
* 節點本身不保存状态，所有数据依赖流程变量

## 使用建議

* Prompt 尽量具体，避免过于模糊的指令
* 明確告诉模型你希望的**輸出格式**
* 对关键流程，建议限制輸出长度或结构，提升稳定性
* 将複雜任務拆分为多个大模型節點，提升可控性

## 典型場景

* 對話回覆生成
* 信息总结与改写
* 结构化数据生成（如 JSON、列表）
* 多步骤推理中的核心思考節點
