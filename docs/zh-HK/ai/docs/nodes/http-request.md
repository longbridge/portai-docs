# HTTP 请求節點

## 是什麼

HTTP 请求節點用于**透過 HTTP/HTTPS 调用外部介面**，获取或提交数据。

它是流程中最常见的**对外通信節點**。

## 您可以用它做什麼

#### 1. 调用外部 API

透過 HTTP 请求節點，你可以：

* 请求第三方服务
* 调用内部业务介面
* 获取实时数据

支援常见 HTTP 方法：

* GET
* POST
* PUT
* DELETE（视平台支援）

---

#### 2. 与业务系统集成

HTTP 请求節點常用于：

* 查询业务数据
* 提交操作指令
* 同步外部系统状态

是 Workflow 和 Agent 的重要基础能力。

---

#### 3. 为模型或 Agent 提供真实数据

你可以将：

* 介面返回结果
* 实时数据

作為后续 LLM 或 Agent 節點的输入，提高结果的准确性和实时性。

## 配置說明

通常需要配置：

* 请求地址（URL）
* 请求方法
* 请求头（Headers）
* 请求参数（Query / Body）

支援使用流程变量作為参数值。

## 输入与輸出

* **输入**：

  * 来自流程变量的请求参数
* **輸出**：

  * 介面返回的响应数据
  * 可用于后续節點处理或判斷

## 使用規則

* HTTP 请求節點只负责请求和返回
* 不做複雜邏輯处理
* 请求失败需要在流程中自行处理（如条件判斷）

## 使用建議

* 介面返回内容尽量标准化，方便后续使用
* 对不稳定介面做好异常和兜底处理
* 敏感信息建议透過安全配置而非明文传递

## 典型場景

* 查询第三方服务
* 调用内部系统介面
* 获取实时或业务数据
* 作為 Agent 可用工具之一
