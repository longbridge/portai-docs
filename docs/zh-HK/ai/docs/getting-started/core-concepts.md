# Agent Builder 的核心基本概念

## Agent 是什麼？

在 Agent Builder 中，**Agent** 是一个可以被執行的智能流程。

它不是一个模型，也不是一段代碼，而是：

> **由多个能力節點組合而成、可以自动运行的 AI 工作流程。**

Agent 可以：

* 理解输入
* 做出判斷
* 调用模型和工具
* 产出结果或回覆


## 两种 Agent 类型：ChatFlow 与 Workflow

Agent Builder 中有两种核心流程类型：

### ChatFlow

* 面向用户對話
* 支援多轮交互
* 輸出透過 **Answer 節點** 返回给用户
* 适合助手、问答、交互式场景

### Workflow

* 面向任務執行
* 强调流程完成与结果产出
* 輸出透過 **End 節點** 返回
* 适合数据生产、自动化任務

**一个流程只能选择其中一种类型。**


## 畫布与節點

### 畫布（Canvas）

畫布是 Agent 的“工作台”：

* 每一个流程都在畫布上完成
* 執行顺序透過節點连接体现
* 流程结构一眼可见

### 節點（Node）

節點是 Agent 的最小能力单元。

每个節點只做一类明確的事情，例如：

* 思考（模型）
* 判斷（条件、分类）
* 執行（工具、介面）
* 控制（循环、分支）
* 輸出（Answer / End）

複雜 Agent = 多个簡單節點的組合。


## LLM 節點 vs Agent 節點

### LLM 節點

* 单次调用模型
* 執行路径固定
* 适合明確、可预期的任務

### Agent 節點

* 目标驱动
* 可多步執行
* 可自主决定是否调用工具

**能用 LLM 的，不用 Agent；
只有路径不确定时，才用 Agent。**


## 变量与上下文

### 变量是什麼？

变量是流程中流动的数据：

* 来自输入
* 来自模型或工具輸出
* 可被后续節點引用

### 上下文是什麼？

上下文是 Agent 執行时的“记忆”：

* 包含当前流程的所有变量
* 在節點之间自动传递
* 决定后续行为和结果


## 显式流程 vs 自主執行

Agent Builder 支援两种構建方式：

### 显式流程

* 使用条件、分支、循环等節點
* 每一步邏輯清晰可控
* 适合高确定性场景

### 自主執行

* 使用 Agent 節點
* 只定义目标和边界
* 适合複雜、不确定任務


## 工具与外部能力

Agent 本身不具备“行动能力”，
它的執行能力来自 **工具**：

* HTTP 介面
* 内部系统能力
* 数据服务

只有接入的工具，Agent 才能使用。


## 輸出的两种方式

### Answer 節點

* 面向用户
* 用于 ChatFlow
* 用来“说话”

### End 節點

* 面向系统
* 用于 Workflow
* 用来“交付结果”

**二者不能同时使用。**


## Agent Builder 的边界认知（很重要）

Agent Builder 适合：

* 智能决策
* 自动化流程
* AI 驱动任務

但它不是：

* 分布式计算框架
* 数据仓库
* 高性能交易系统


## 一句话总模型

> **Agent Builder =
> 用流程的方式，组织模型、工具和邏輯，
> 構建可執行、可演进的 AI Agent。**
