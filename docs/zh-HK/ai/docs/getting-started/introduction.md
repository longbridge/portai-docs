# Agent Builder 平台整体介紹

## 什么是 Agent Builder？

**Agent Builder** 是 Infra 平台提供的一套 **視覺化、低代碼的 AI Agent 構建工具**。

透過畫布化流程编辑的方式，你可以：

* 将大模型能力、工具能力与业务邏輯組合起来
* 快速構建可运行的 AI Agent
* 用于對話场景（ChatFlow）或任務型场景（Workflow）

無需从零編寫複雜代碼，即可搭建可维护、可迭代的智能應用。


## 平台能做什么？

### 視覺化構建 AI Agent

Agent Builder 提供 **視覺化畫布 + Low Code** 的方式来搭建 Agent：

* 透過節點拖拽，組合完整流程
* 每个節點代表一种明確能力（模型、工具、判斷、執行等）
* 流程结构清晰，可讀性强

即使是複雜的 AI 邏輯，也可以被拆解为清晰的流程图。

### 支援两类核心 Agent 形態

#### ChatFlow（對話型 Agent）

適用于：

* 智能问答
* 助手型應用
* 多轮對話场景

特点：

* 面向用户实时輸出回覆
* 支援上下文理解与多轮交互
* 以 **Answer 節點** 作為对外輸出

---

#### Workflow（任務型 Agent）

適用于：

* 数据生产
* 自动化任務
* 系统间流程编排

特点：

* 强调流程執行与结果产出
* 可调用模型、工具、介面
* 以 **End 節點** 作為流程出口

### 丰富的節點能力，覆盖完整 Agent 生命週期

Agent Builder 内置多种能力節點，可覆盖从输入到輸出的完整链路，例如：

* 大模型推理与生成
* Agent 自主决策与工具调用
* 条件判斷与分支控制
* 参数提取、问题分类
* HTTP 介面调用
* 自定义代碼处理
* 变量管理、模板转换
* 迭代、循环与分支聚合

你可以根据业务複雜度，自由选择“显式流程”或“Agent 自主執行”的方式。

### 支援多种應用场景

Agent Builder 適用于多种 AI 應用场景，包括但不限于：

* 投研与分析类助手
* 事件驱动的策略与流程
* 知识问答与内部助手
* 数据查询与自动化处理
* 智能决策支援系统

同一套平台，可以支撑不同複雜度、不同形態的 AI 應用。


## 相比传统开发方式的优势

### 流程視覺化，降低理解与协作成本

* AI 邏輯以流程图形式呈现
* 非技术角色也能理解整体邏輯
* 方便产品、研发、业务多方协作

### 内置通用能力，提升开发效率

* 常见 AI 能力已被封装为節點
* 無需重复造轮子
* 相比纯代碼方式，上手和迭代更快

### 更容易接入和升级 AI 能力

* 平台可持续接入新的模型能力
* Agent 邏輯無需大幅改动即可升级
* 更适合 AI 技术快速演进的现状


## 平台的边界与限制

### 不适合的场景

Agent Builder **不是** 用来替代所有系统：

* 不适合大规模分布式计算
* 不适合作為离线数据仓库或数据湖
* 不用于高频、超低延迟的核心交易链路

它更适合 **智能决策、自动化流程与 AI 驱动任務**。

### 能力边界依赖外部接入

Agent 的能力上限，取决于可用的外部能力：

* 行情、资讯、业务数据
* 内部系统介面
* 第三方服务能力

这些能力需要透過标准化方式接入后，Agent 才能使用。

### 使用规模需合理评估

* 在大规模并发或高频调用前
* 需要评估執行成本与稳定性
* 对关键业务流程，建议提前进行容量与风险评估


## 适合什么样的人使用？

Agent Builder 特别适合：

* 希望快速验证 AI 想法的产品与业务团队
* 希望降低 AI 應用开发门槛的研发团队
* 需要構建可维护、可解释 AI 流程的场景

你不需要从“写一个 AI 系统”開始，
而是从 **“设计一个 Agent 的工作方式”** 開始。

> 它让 AI 應用从“难以维护的代碼”，
> 变成“清晰、可組合、可演进的智能流程”。
