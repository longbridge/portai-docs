# 管理知識庫

本文說明如何在知識庫中管理文件、分段及知識庫設定。

## 管理文件

每個匯入的本機檔案在知識庫中以「文件」形式存在。你可在文件列表中檢視和管理所有文件，按需保留或移除，並根據上傳時間判斷時效性。

文件列表支援以下操作：

| 操作     | 說明 |
|----------|------|
| 新增檔案 | 在文件列表中點擊「新增檔案」進入上傳流程，可選擇檔案、設定分段並檢視處理進度。分段設定目前僅能在上傳或重新上傳時修改，文件管理中暫不支援單一文件分段設定編輯。 |
| 刪除文件 | 永久刪除該文件。不可撤銷、不可恢復。 |
| 啟用/停用 | 暫時將該文件納入或排除在檢索範圍之外。 |
| 編輯     | 進入文件詳情，對分段內容進行編輯。詳見下文「管理分段」。 |
| 重新命名   | 修改該文件的顯示名稱。 |

## 管理分段

文件按分段設定被拆分為一個或多個「分段」，分段是檢索的基本單位。你可在文件的分段列表中檢視和管理分段，以提升檢索效率與準確性。

分段列表支援以下操作：

| 操作           | 說明 |
|----------------|------|
| 新增分段       | 在文件詳情頁新增分段，可同時設定分段關鍵詞，便於後續檢索匹配。 |
| 刪除分段       | 永久刪除該分段。不可撤銷、不可恢復。 |
| 啟用/停用      | 暫時將該分段納入或排除檢索。可透過分段前的圓圈批量勾選後操作。已停用的分段仍可編輯。 |
| 新增/編輯/刪除關鍵詞 | 為分段維護關鍵詞，提升該分段的可檢索性。 |

## 分段設定實務建議

分段完成後，建議逐段檢查：語義完整、未被強制截斷、長度適中，以保證檢索準確性與相關性。

常見問題與影響：

| 情況         | 影響 |
|--------------|------|
| 分段過短     | 上下文不足，易造成語義遺失和回答不準確。 |
| 分段過長     | 易包含無關內容，引入雜訊、降低檢索精度。 |
| 語義不完整   | 句子或段落被強行切斷，易導致檢索結果遺漏或誤導。 |

## 知識庫設定

在知識庫頁點擊右上角「知識庫設定」進入設定頁。

支援的操作包括：

| 操作           | 說明 |
|----------------|------|
| 修改名稱與描述 | 自訂知識庫的名稱與描述，用於識別與區分。名稱不可與其他知識庫重複，否則修改將失敗。 |
| 修改檢索設定與模型 | 調整知識庫的檢索方式及所用模型、參數。具體選項與含義參見「建立知識庫」中關於檢索策略的說明。 |
