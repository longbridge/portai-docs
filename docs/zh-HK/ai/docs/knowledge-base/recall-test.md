# 召回測試

透過召回測試，可模擬用戶提問，驗證知識庫的召回效果，並嘗試不同檢索設定以找到最佳配置。

## 進入召回測試

在**知識庫文件列表頁**或**知識庫設定頁**，點擊右上角「召回測試」圖示，進入測試頁面。

## 使用說明

| 項目       | 說明 |
|------------|------|
| 測試方式   | 輸入問題，檢視系統從知識庫中召回的文字分段及排序結果。 |
| 設定作用範圍 | 在召回測試中修改的檢索設定，僅在目前測試工作階段中暫時生效，用於對比不同檢索策略與參數的效果。檢索參數含義見「建立知識庫」中的檢索策略說明。 |
| 歷史記錄   | 本頁會記錄該知識庫相關的召回測試記錄，包括檢索方式、查詢文字及時間。支援點擊歷史問句快速複製，便於重複測試或對比。 |

## 建議用法

- 用業務中的典型問題做測試，觀察召回片段是否涵蓋所需資訊。
- 在測試中切換不同檢索策略（向量、全文、混合）及 TopK、Score 閾值等參數，對比召回品質與數量。
- 結合歷史記錄，對同一問題在不同設定下的結果進行對比，確定適合該知識庫的配置。
