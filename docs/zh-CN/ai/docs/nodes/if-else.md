# If-Else

## 是什么
条件判断节点用于**根据已有变量进行判断，并决定流程接下来走哪一条路径**。
它让流程具备“选择”和“分支”能力，而不是只能线性执行。

## 你可以用它做什么

#### 1. 根据结果走不同流程

你可以基于上游节点的输出：

* 判断文本内容
* 判断变量是否存在
* 判断数值大小或状态

根据判断结果，让流程进入不同的分支。

例如：

* 判断用户问题类型
* 判断模型是否返回有效结果
* 判断是否需要调用工具

---

#### 2. 构建多分支流程

条件判断节点可以：

* 配置多个判断条件
* 每个条件对应一条分支路径
* 未命中条件时走默认分支

这让你可以构建更复杂、更灵活的流程。

## 条件配置说明

* 条件通常基于流程变量进行判断
* 支持常见判断方式，如：

  * 等于 / 不等于
  * 包含 / 不包含
  * 是否为空
  * 数值比较（大于、小于等）

每个条件都会连接到一个下游节点。

## 使用规则

* 条件判断节点必须有至少一个下游分支
* 每条分支都只能在条件满足时才会被执行
* 条件本身不产生新数据，只负责“判断和分流”

## 使用建议

* 条件尽量清晰，避免多个条件重叠
* 为复杂判断设置一个兜底分支，防止流程中断
* 在进入条件判断前，确保相关变量已准备好

## 典型场景

* 根据用户意图选择不同处理流程
* 判断是否需要再次调用模型
* 判断工具调用是否成功
* 多轮流程中的决策点
