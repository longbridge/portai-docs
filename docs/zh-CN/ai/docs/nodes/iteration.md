# 迭代节点（Iteration）

## 是什么

迭代节点用于**对一组数据进行逐项处理**。
它会把一份“集合型数据”（如数组、列表）拆分成多个元素，**按顺序重复执行同一段子流程**。

你可以把它理解为：

> **“对列表里的每一项，都走一遍同样的流程”**

## 你可以用它做什么

#### 1. 批量处理数据

迭代节点非常适合：

* 遍历列表
* 批量生成结果
* 对每一项执行相同逻辑

例如：

* 对多个问题分别生成答案
* 对搜索结果逐条分析
* 对一组数据做格式转换

---

#### 2. 将单个流程复用到多项数据

你只需要设计一次处理逻辑，
迭代节点会帮你自动重复执行。

## 工作方式（理解即可）

1. 输入一个列表类型的变量
2. 每次取出其中一个元素
3. 执行连接在迭代节点内的子流程
4. 收集每一次执行的结果
5. 输出为新的结果集合

## 输入与输出

* **输入**：

  * 列表 / 数组类型的数据
* **输出**：

  * 每一轮执行产生的结果集合

## 使用规则

* 迭代节点必须接收集合类型输入
* 子流程中不要修改原始集合结构
* 每一项迭代逻辑应保持一致

## 使用建议

* 单次迭代逻辑尽量简单
* 避免在迭代中嵌套复杂 Agent
* 对大规模列表注意执行成本

## 典型场景

* 批量内容生成
* 批量分析与处理
* 列表型数据的逐项执行
