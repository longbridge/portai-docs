# 变量赋值器

## 是什么

变量赋值器用于**创建、修改或覆盖流程中的变量值**。
它让你可以在流程执行过程中，**显式地控制变量内容**，而不是依赖模型或工具的隐式输出。

你可以把它理解为：

> **“在流程中手动设置或更新变量的地方。”**

## 你可以用它做什么

#### 1. 创建新变量

你可以使用变量赋值器：

* 定义新的流程变量
* 为变量设置初始值或默认值
* 将计算结果保存为可复用的数据

例如：

* 设置状态标识
* 初始化计数器
* 生成中间结果变量

---

#### 2. 修改已有变量

变量赋值器可以：

* 覆盖上游节点产生的变量
* 基于已有变量生成新值
* 在不同分支中设置不同变量状态

这在条件分支或循环中非常常见。

---

#### 3. 控制流程状态

通过变量赋值器，你可以：

* 标记流程当前状态
* 控制循环或条件判断的依据
* 在流程中显式表达“进度”或“结果状态”

## 输入与输出

* **输入**：

  * 可引用任意已有流程变量
* **输出**：

  * 更新后的变量集合
  * 变量可直接被后续节点使用

## 使用规则

* 变量名必须唯一（同一作用域内）
* 新赋值会覆盖同名变量的旧值
* 赋值器本身不进行复杂逻辑推理

## 使用建议

* 用变量赋值器做**明确、可控的状态修改**
* 避免在模型输出中隐式改变关键变量
* 关键流程变量建议集中在赋值器中修改，方便排查问题
* 变量命名尽量语义清晰，避免歧义

## 典型场景

* 设置流程执行状态（如 success / failed）
* 在条件分支中标记不同结果
* 循环中更新计数或累计结果
* 将复杂节点输出整理为统一变量

## 什么时候不该用？

* 只需要临时使用某个节点输出
* 不关心变量的长期复用
* 需要复杂计算或推理（应使用 Code 或 LLM）

## 与其他节点的区别（快速理解）

| 节点      | 主要作用        |
| ------- | ----------- |
| 变量赋值器   | 显式设置 / 修改变量 |
| Code 节点 | 执行确定性计算     |
| LLM 节点  | 推理与生成       |
| 参数提取器   | 从文本中提取变量    |

## 一句话记住

> **变量赋值器不是“计算”，而是“控制”。**
> 用它来让流程的状态和数据变化**看得见、管得住**。
