# 工具节点（Tool）

## 是什么

工具节点用于**调用外部能力或系统功能**，让流程不仅能“思考”，还能“行动”。

工具可以是：

* 接口（API）
* 内部服务
* 数据处理函数
* 业务系统能力

## 你可以用它做什么

#### 1. 调用外部接口

通过工具节点，你可以：

* 查询数据
* 写入或更新信息
* 调用第三方或内部系统接口

例如：

* 查询用户信息
* 获取实时数据
* 提交业务请求

---

#### 2. 执行确定性操作

与大模型不同，工具节点：

* 输出结果稳定、可预测
* 适合执行明确、规则化的任务
* 常用于补充模型能力

---

#### 3. 与大模型协作

常见用法是：

1. 由大模型判断是否需要调用工具
2. 将判断结果传入工具节点
3. 将工具返回结果再交给大模型处理

这种组合可以显著提升流程的实用性。

## 输入与输出

* **输入**：

  * 来自开始节点或上游节点的变量
* **输出**：

  * 工具执行后的返回结果
  * 可作为后续节点的输入或判断条件

## 使用规则

* 工具节点必须配置具体的工具或接口
* 输入参数需与工具定义保持一致
* 若工具执行失败，应通过条件节点或兜底流程处理

## 使用建议

* 工具职责尽量单一，避免复杂逻辑堆在一个工具中
* 对关键工具调用结果进行判断，避免流程异常中断
* 将“决策”交给模型，将“执行”交给工具

## 典型场景

* 查询数据库或业务系统
* 调用搜索、计算、转换类能力
* 执行业务操作（下单、更新状态等）
* 获取实时或结构化数据
