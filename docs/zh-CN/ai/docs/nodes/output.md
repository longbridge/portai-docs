# 结束节点（End）

## 是什么

结束节点用于 Workflow 的 **标记流程的完成位置**。
当流程执行到结束节点时，整个流程会停止运行，并将最终结果返回给调用方或用户。

这是流程中的**终点节点**。

## 你可以用它做什么

#### 1. 结束流程执行

结束节点用于明确告诉系统：

* 当前流程已经完成
* 不再继续执行后续节点

无论流程多复杂，最终都应以结束节点作为收尾。

---

#### 2. 返回最终输出结果

你可以在结束节点中：

* 选择需要返回的变量或内容
* 将上游节点（如大模型节点）的输出作为最终结果返回

例如：

* 返回模型生成的文本
* 返回结构化数据（如 JSON）
* 返回经过多步处理后的结果

---

#### 3. 统一流程出口

在包含条件分支的流程中：

* 不同分支可以连接到同一个结束节点
* 用于统一流程的最终输出和结束行为

## 输出说明

* 结束节点的输出即为**流程的最终输出**
* 返回内容会：

  * 展示给用户
  * 或作为 API 调用的响应结果
* 如果未配置返回内容，流程将只执行但不返回数据

## 使用规则

* 一个流程中**至少需要一个结束节点**
* 结束节点：

  * 不能连接下游节点
  * 只能作为流程的终点
* 一个流程中可以存在**多个结束节点**（用于不同分支返回不同结果）

## 使用建议

* 明确指定你希望返回的内容，避免返回无关变量
* 对外提供 API 的流程，建议返回结构清晰、稳定的数据格式
* 多分支流程中，确保每个分支最终都能走到某个结束节点

## 典型场景

* 返回对话回复给用户
* 返回 Agent 执行结果
* 返回工具或模型处理后的最终数据
* 作为子流程的结束出口
