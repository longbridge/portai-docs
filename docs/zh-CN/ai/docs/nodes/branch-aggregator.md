# 分支聚合器

## 是什么

分支聚合器用于**将来自不同分支的结果重新汇合到一起**，让流程在分支之后可以继续统一执行。

你可以把它理解为：

> **“流程的合流点。”**

## 你可以用它做什么

#### 1. 合并不同分支的执行结果

在条件判断或分类之后：

* 不同分支可以产生不同结果
* 分支聚合器负责接收这些结果
* 将流程重新拉回到一条主线

---

#### 2. 统一后续处理逻辑

使用分支聚合器后：

* 不需要在每个分支重复配置相同节点
* 后续节点只需连接一次

这能显著减少流程重复。

## 输入与输出

* **输入**：

  * 来自不同分支的变量或结果
* **输出**：

  * 聚合后的统一结果
  * 可被后续节点使用

## 使用规则

* 所有需要汇合的分支都应连接到同一个聚合器
* 聚合器本身不做判断
* 聚合规则需明确（如选择某个结果、合并字段等）

## 使用建议

* 分支输出结构尽量保持一致
* 聚合前避免分支变量命名冲突
* 聚合器后统一做模板转换或输出

## 典型场景

* 条件分支后的统一输出
* 多路径处理后的结果汇总
* 分类流程结束后的统一回复
