# HTTP 请求节点

## 是什么

HTTP 请求节点用于**通过 HTTP/HTTPS 调用外部接口**，获取或提交数据。

它是流程中最常见的**对外通信节点**。

## 你可以用它做什么

#### 1. 调用外部 API

通过 HTTP 请求节点，你可以：

* 请求第三方服务
* 调用内部业务接口
* 获取实时数据

支持常见 HTTP 方法：

* GET
* POST
* PUT
* DELETE（视平台支持）

---

#### 2. 与业务系统集成

HTTP 请求节点常用于：

* 查询业务数据
* 提交操作指令
* 同步外部系统状态

是 Workflow 和 Agent 的重要基础能力。

---

#### 3. 为模型或 Agent 提供真实数据

你可以将：

* 接口返回结果
* 实时数据

作为后续 LLM 或 Agent 节点的输入，提高结果的准确性和实时性。

## 配置说明

通常需要配置：

* 请求地址（URL）
* 请求方法
* 请求头（Headers）
* 请求参数（Query / Body）

支持使用流程变量作为参数值。

## 输入与输出

* **输入**：

  * 来自流程变量的请求参数
* **输出**：

  * 接口返回的响应数据
  * 可用于后续节点处理或判断

## 使用规则

* HTTP 请求节点只负责请求和返回
* 不做复杂逻辑处理
* 请求失败需要在流程中自行处理（如条件判断）

## 使用建议

* 接口返回内容尽量标准化，方便后续使用
* 对不稳定接口做好异常和兜底处理
* 敏感信息建议通过安全配置而非明文传递

## 典型场景

* 查询第三方服务
* 调用内部系统接口
* 获取实时或业务数据
* 作为 Agent 可用工具之一
