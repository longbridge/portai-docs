# 模板转换

## 是什么

模板转换节点用于**按照预设模板，将变量和内容转换成指定格式的输出**。

它不负责推理，也不做业务判断，只做一件事：

> **“把已有内容，整理成你想要的样子。”**

## 你可以用它做什么

#### 1. 统一输出格式

模板转换可以：

* 将多个变量组合成一段文本
* 输出统一格式的内容
* 保证不同分支的结果结构一致

例如：

```text
标题：{{title}}
内容：{{content}}
总结：{{summary}}
```

---

#### 2. 生成结构化结果

你可以使用模板：

* 生成 JSON
* 生成固定字段的文本结构
* 为 API 或下游系统准备标准化数据

---

#### 3. 解耦“内容生成”和“内容展示”

常见做法是：

* 用 LLM / Agent 负责生成内容
* 用模板转换负责整理和输出

这样流程更清晰，也更易维护。

## 输入与输出

* **输入**：

  * 任意流程变量
* **输出**：

  * 按模板生成的内容
  * 可作为 Answer 或 End 的输入

## 使用规则

* 模板中引用的变量必须已存在
* 模板转换不修改变量值，只生成新内容
* 不进行条件判断或逻辑处理

## 使用建议

* 模板尽量简单、清晰
* 复杂逻辑不要写在模板中
* 对外输出建议统一经过模板转换

## 典型场景

* 对话回复整理
* API 返回结构统一
* 多分支结果格式对齐
