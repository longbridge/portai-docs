# 开始

## 是什么

开始节点是**流程的起点**。
当流程被触发时，系统会从这里开始执行，并把你提供的输入内容传递给后续节点。

每个流程**必须**包含一个开始节点。

## 你可以用它做什么

#### 1. 接收流程输入

开始节点用于接收流程运行时的输入内容，例如：

* 用户输入的文本
* 接口调用时传入的参数
* 上层系统传递的数据

这些输入会作为流程的初始数据，供后续所有节点使用。

---

#### 2. 定义流程可用的变量

你可以在开始节点中：

* 定义流程的输入字段
* 设置字段名称和类型
* 作为后续节点引用的数据来源

例如：

* `query`：用户的问题
* `user_id`：用户标识
* `context`：上下文信息

---

#### 3. 触发整个流程运行

当流程被调用或执行时：

* 系统会从开始节点启动
* 按照画布上的连接顺序，依次执行后续节点
* 开始节点本身不进行处理，只负责“开始”和“传递”

## 使用规则

* 一个流程中**只能有一个开始节点**
* 开始节点：

  * 不能被其他节点连接
  * 必须连接至少一个下游节点
* 如果没有开始节点，流程无法运行

## 使用建议

* 将**所有流程必需的输入**都放在开始节点中定义
* 字段命名尽量清晰，方便后续节点理解和引用
* 不要在开始节点中承担逻辑处理，它只负责输入和启动

## 典型场景

* 对话流程的用户输入入口
* Agent 执行前的参数配置
* API 流程的请求参数起点
* 子流程的统一入口
