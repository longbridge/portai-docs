# 单节点调试

## 功能说明

Agent Builder 支持对 **单个节点进行独立调试**，
在不发布 Agent 的情况下，验证某一个节点的输入、执行逻辑和输出结果。

单节点调试主要用于：

* 快速验证节点配置是否正确
* 排查流程中某一步的执行问题
* 调试复杂流程中的关键节点


## 核心特点

* **无需发布 Agent**
* **不依赖完整流程执行**
* 仅针对当前选中的节点进行调试
* 使用当前画布中可见的最新配置


## 适合的使用场景

* 调试大模型节点的 Prompt 是否符合预期
* 验证 HTTP / 工具节点的返回结果
* 检查参数提取、变量赋值是否正确
* 在修改流程前，对关键节点进行验证


## 使用建议

* 在流程较复杂时，优先通过单节点调试定位问题
* 节点配置调整后，及时重新调试确认效果
* 单节点调试通过后，再进行整体流程调试


## 一句话总结

> **单节点调试用于“验证一步”，
> 是定位问题和快速修正的首选方式。**
