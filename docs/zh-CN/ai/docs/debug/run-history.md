# 运行记录与试运行结果说明

## 试运行的基本原则

在 Agent Builder 中，**试运行用于调试和验证流程行为**，不等同于正式运行或发布后的执行。

所有试运行均遵循以下原则：

* **试运行记录不予保存**
* 仅用于当前调试和观察
* 结果以界面实时展示为准


## ChatFlow 的试运行记录

### 展示方式

* ChatFlow 的试运行结果会以 **聊天窗口的形式展示**
* 内容与真实对话体验一致
* 支持多轮交互展示

## 记录规则

* 聊天内容 **仅在当前试运行中可见**
* 试运行结束后：

  * 聊天记录不会被保存
  * 不会作为历史对话留存


## Workflow 的试运行记录

### 展示方式

* Workflow 的试运行结果以 **变量输出的形式展示**
* 可查看流程执行后各关键变量的值

## 记录规则

* 变量输出仅用于当前试运行查看
* 试运行结束后：

  * 变量结果不会被保存
  * 不会生成可追溯的运行记录


## 单节点试运行记录

### 展示方式

* 单节点试运行结果同样以 **变量输出的形式展示**
* 用于观察该节点的输入与输出结果

## 记录规则

* 单节点试运行结果不予保存
* 每次试运行互不影响


## 运行过程可视化

在试运行过程中：

* 可以观察到 **节点的运行过程**
* 节点的执行状态会在画布中体现
* 有助于理解流程执行顺序和数据流转

具体的展示方式与细节，请以界面中的实际表现为准。


## 使用提示

* 试运行适合用于调试与验证，不适合用于留存结果
* 关键逻辑建议在试运行通过后再进行发布
* 若需要可追溯的执行结果，请使用已发布 Agent 的正式运行能力


## 一句话总结

> **试运行只用于“看过程、验结果”，
> 不用于“存记录、做追溯”。**
> 所有展示内容请以当前界面为准。
