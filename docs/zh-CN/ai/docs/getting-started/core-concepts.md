# Agent Builder 的核心基本概念

## Agent 是什么？

在 Agent Builder 中，**Agent** 是一个可以被执行的智能流程。

它不是一个模型，也不是一段代码，而是：

> **由多个能力节点组合而成、可以自动运行的 AI 工作流程。**

Agent 可以：

* 理解输入
* 做出判断
* 调用模型和工具
* 产出结果或回复


## 两种 Agent 类型：ChatFlow 与 Workflow

Agent Builder 中有两种核心流程类型：

### ChatFlow

* 面向用户对话
* 支持多轮交互
* 输出通过 **Answer 节点** 返回给用户
* 适合助手、问答、交互式场景

### Workflow

* 面向任务执行
* 强调流程完成与结果产出
* 输出通过 **End 节点** 返回
* 适合数据生产、自动化任务

**一个流程只能选择其中一种类型。**


## 画布与节点

### 画布（Canvas）

画布是 Agent 的“工作台”：

* 每一个流程都在画布上完成
* 执行顺序通过节点连接体现
* 流程结构一眼可见

### 节点（Node）

节点是 Agent 的最小能力单元。

每个节点只做一类明确的事情，例如：

* 思考（模型）
* 判断（条件、分类）
* 执行（工具、接口）
* 控制（循环、分支）
* 输出（Answer / End）

复杂 Agent = 多个简单节点的组合。


## LLM 节点 vs Agent 节点

### LLM 节点

* 单次调用模型
* 执行路径固定
* 适合明确、可预期的任务

### Agent 节点

* 目标驱动
* 可多步执行
* 可自主决定是否调用工具

**能用 LLM 的，不用 Agent；
只有路径不确定时，才用 Agent。**


## 变量与上下文

### 变量是什么？

变量是流程中流动的数据：

* 来自输入
* 来自模型或工具输出
* 可被后续节点引用

### 上下文是什么？

上下文是 Agent 执行时的“记忆”：

* 包含当前流程的所有变量
* 在节点之间自动传递
* 决定后续行为和结果


## 显式流程 vs 自主执行

Agent Builder 支持两种构建方式：

### 显式流程

* 使用条件、分支、循环等节点
* 每一步逻辑清晰可控
* 适合高确定性场景

### 自主执行

* 使用 Agent 节点
* 只定义目标和边界
* 适合复杂、不确定任务


## 工具与外部能力

Agent 本身不具备“行动能力”，
它的执行能力来自 **工具**：

* HTTP 接口
* 内部系统能力
* 数据服务

只有接入的工具，Agent 才能使用。


## 输出的两种方式

### Answer 节点

* 面向用户
* 用于 ChatFlow
* 用来“说话”

### End 节点

* 面向系统
* 用于 Workflow
* 用来“交付结果”

**二者不能同时使用。**


## Agent Builder 的边界认知（很重要）

Agent Builder 适合：

* 智能决策
* 自动化流程
* AI 驱动任务

但它不是：

* 分布式计算框架
* 数据仓库
* 高性能交易系统


##  一句话总模型

> **Agent Builder =
> 用流程的方式，组织模型、工具和逻辑，
> 构建可执行、可演进的 AI Agent。**
