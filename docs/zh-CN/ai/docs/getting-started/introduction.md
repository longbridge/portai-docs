# Agent Builder 平台整体介绍

## 什么是 Agent Builder？

**Agent Builder** 是 Infra 平台提供的一套 **可视化、低代码的 AI Agent 构建工具**。

通过画布化流程编辑的方式，你可以：

* 将大模型能力、工具能力与业务逻辑组合起来
* 快速构建可运行的 AI Agent
* 用于对话场景（ChatFlow）或任务型场景（Workflow）

无需从零编写复杂代码，即可搭建可维护、可迭代的智能应用。


## 平台能做什么？

### 可视化构建 AI Agent

Agent Builder 提供 **可视化画布 + Low Code** 的方式来搭建 Agent：

* 通过节点拖拽，组合完整流程
* 每个节点代表一种明确能力（模型、工具、判断、执行等）
* 流程结构清晰，可读性强

即使是复杂的 AI 逻辑，也可以被拆解为清晰的流程图。

### 支持两类核心 Agent 形态

#### ChatFlow（对话型 Agent）

适用于：

* 智能问答
* 助手型应用
* 多轮对话场景

特点：

* 面向用户实时输出回复
* 支持上下文理解与多轮交互
* 以 **Answer 节点** 作为对外输出

#### Workflow（任务型 Agent）

适用于：

* 数据生产
* 自动化任务
* 系统间流程编排

特点：

* 强调流程执行与结果产出
* 可调用模型、工具、接口
* 以 **End 节点** 作为流程出口

### 丰富的节点能力，覆盖完整 Agent 生命周期

Agent Builder 内置多种能力节点，可覆盖从输入到输出的完整链路，例如：

* 大模型推理与生成
* Agent 自主决策与工具调用
* 条件判断与分支控制
* 参数提取、问题分类
* HTTP 接口调用
* 自定义代码处理
* 变量管理、模板转换
* 迭代、循环与分支聚合

你可以根据业务复杂度，自由选择“显式流程”或“Agent 自主执行”的方式。

### 支持多种应用场景

Agent Builder 适用于多种 AI 应用场景，包括但不限于：

* 投研与分析类助手
* 事件驱动的策略与流程
* 知识问答与内部助手
* 数据查询与自动化处理
* 智能决策支持系统

同一套平台，可以支撑不同复杂度、不同形态的 AI 应用。


## 相比传统开发方式的优势

### 流程可视化，降低理解与协作成本

* AI 逻辑以流程图形式呈现
* 非技术角色也能理解整体逻辑
* 方便产品、研发、业务多方协作

### 内置通用能力，提升开发效率

* 常见 AI 能力已被封装为节点
* 无需重复造轮子
* 相比纯代码方式，上手和迭代更快

### 更容易接入和升级 AI 能力

* 平台可持续接入新的模型能力
* Agent 逻辑无需大幅改动即可升级
* 更适合 AI 技术快速演进的现状


## 平台的边界与限制

### 不适合的场景

Agent Builder **不是** 用来替代所有系统：

* 不适合大规模分布式计算
* 不适合作为离线数据仓库或数据湖
* 不用于高频、超低延迟的核心交易链路

它更适合 **智能决策、自动化流程与 AI 驱动任务**。

### 能力边界依赖外部接入

Agent 的能力上限，取决于可用的外部能力：

* 行情、资讯、业务数据
* 内部系统接口
* 第三方服务能力

这些能力需要通过标准化方式接入后，Agent 才能使用。

### 使用规模需合理评估

* 在大规模并发或高频调用前
* 需要评估执行成本与稳定性
* 对关键业务流程，建议提前进行容量与风险评估


## 适合什么样的人使用？

Agent Builder 特别适合：

* 希望快速验证 AI 想法的产品与业务团队
* 希望降低 AI 应用开发门槛的研发团队
* 需要构建可维护、可解释 AI 流程的场景

你不需要从“写一个 AI 系统”开始，
而是从 **“设计一个 Agent 的工作方式”** 开始。


## 一句话总结

> **Agent Builder 是一个用流程思维构建 AI Agent 的平台。**
> 它让 AI 应用从“难以维护的代码”，
> 变成“清晰、可组合、可演进的智能流程”。
