# 工作空间用户管理

## 工作空间是什么？

工作空间是 Agent 的**组织和协作单位**：

* Agent 必须隶属于某一个工作空间
* 用户通过工作空间进行协作
* 权限控制以工作空间为边界


## 工作空间角色说明

每个工作空间中包含以下角色：

### Owner（所有者）

* 创建该工作空间的用户
* 每个人默认拥有一个自己的工作空间
* 拥有最高权限

Owner 可以：

* 管理工作空间成员
* 邀请其他用户加入
* 管理该空间下的所有 Agent

## Editor（编辑者）

* 被 Owner 邀请加入工作空间
* 可以编辑该空间下的 Agent

Editor 可以：

* 编辑 Agent 流程
* 调试和试运行 Agent
* 发布 Agent（以平台实际权限为准）

## Viewer（查看者）

* 被 Owner 邀请加入工作空间
* 仅拥有使用权限

Viewer 可以：

* 使用 Agent
* 查看 Agent 的运行结果
* 不能编辑 Agent


## 用户与工作空间的关系

* 每个人都有 **自己的工作空间（作为 Owner）**
* 每个人也可以被 **邀请加入多个其他工作空间**
* 不同工作空间中的角色和权限彼此独立


## 使用提示

* 建议通过角色区分“编辑”和“使用”职责
* 核心 Agent 的编辑权限应谨慎分配
* 多人协作时注意发布与版本管理


## 一句话总结

> **工作空间定义了 Agent 的归属与协作边界，
> 角色决定你在这个空间中能做什么。**
