# Personal Settings - Workspace Invitation

## Function Description

Workspace invitations are used to **add other users to your workspace** for multi-person collaboration.

Only the **Owner** of a workspace can initiate invitations.


## Invitation Methods and Results

* Owners can invite other users into the workspace
* Invited users will join with a specified role:
  * Editor
  * Viewer

After a successful invitation:

* The invited user can see the workspace in their own account
* They can perform operations based on their role permissions


## Multi-workspace Support

* A user can belong to **multiple workspaces** simultaneously
* In different workspaces:
  * They can have different roles
  * They can manage different Agents

Users can freely switch between workspaces.


## Permission Description

* Invitations only affect the current workspace
* Does not affect the user's personal workspace
* Does not affect permissions in other workspaces


## Usage Recommendations

* Clarify collaboration goals and role division before inviting
* Editing permissions should only be granted to members actually involved in building
* Use workspaces to distinguish between different projects or teams
